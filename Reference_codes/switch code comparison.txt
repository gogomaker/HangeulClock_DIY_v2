기존 스위치 인식 코드와 새로 작성할 코드의 차이점

**스위치 코드의 숫자값 의미
0. none  (스위치 값 없음)
1. short (짧게 눌림)
2. long  (길게 눌림)

스위치 순서
0. 모드스위치
1. LED스위치
2. 시간스위치
3. 알람스위치

우선, 기존 코드의 방식은 이렇다.

스위치 인식함수{
    상태 읽음
    디바운싱 작업 후...
    진짜로 버튼이 눌린 거라면{
        버튼의 상태가 전과 다르다면 {
            버튼 상태 기록;
            버튼이 눌렸다면 {
                버튼 눌린 시간 체크
            } 
            버튼이 때졌다면 {
                버튼이 짧게 눌렸다면 {
                    짧게 눌린 행동 수행
                }
                버튼이 길게 눌렸으면 {
                    길게 눌린 행동 수행
                }
            }
        }
    }
}

대충 이런 식인데, 문제점이 몇 가지 있음
1. 메인 펑션의 변수를 직접 편집함.
2. 너무 if문의 깊이가 깊음
3. 함수가 void임(리턴이 없음)

그래서, 새로이 하는 함수는 위의 문제를 개선하고자 함.

int 스위치 인식함수 {
    상태를 읽음
    디바운싱 값 입력
    바운싱이라면 { 거름 }
    스위치 상태가 다르지 않으면 { 거름 }

}
